DMX 舞台照明制御システム

使用技術

  • Arduino
    • C++
  • Python3
    • PySerial
    • Pygame

製作期間

2023年1月～2023年12月

概要

私は、大学在学中に所属していた軽音楽部で、1年間照明係を担当しました。このとき、これまで当部が学内でライブを行う際に使用していた舞台照明のシステムを一部更新し、その表現力を高める/利便性の向上に成功しました。更新部分について、使用した電子回路の一部を自力で作成したほか、電子制御・フロントエンド双方のプログラムも一から作成しました。

技術構成図

頑張ったこと

保守性・運用性の向上(ソフトウェア)

このシステムの作成にあたり、「自分の手を離れても運用・保守が容易に行える、という状況にする」ことを心がけました。

なぜなら、本ソリューションにおける大きな懸念事項の1つに「1年後、自分は照明係を降りることになり(※照明係の任期は1年で、担当学年は固定)、作成したシステムは自分の手を離れて運用されることになる」ということがありました。
最初に作ったシステムは、1～2ヶ月程度で運用可能な状態にまで持っていくことができましたが、そのシステムは以下のような問題があり、このままの状態で次世代には引き継げないと判断しました:

  1. フロントエンドがHot Soup Processor 3という(あまりメジャーではない)言語で作成されており、後任がプログラムの読解・編集ができる可能性が低い。
  2. 照明機材の数・構成・チャンネル配置がハードコーディングされており、運用性が低い。
  3. HSP3 は自動ビルド性が低い。手作業でのインストールとビルドをしなければならない。若しくは自動ビルドツールはアンチウイルスに誤って検疫されることが多く、自動ビルドには使えない。

上記の問題を 対処/影響を軽減 するために、それぞれ以下のことを気をつけながら、改良版のアプリケーションを作成しました:

  1. フロントエンドは Python3 + Pygame で一から書き直した。Python であればクロスプラットフォーム性と後方互換性をある程度確保できるし、読み書きできる大学生も多い。
  2. 機器の数・種類・チャンネル割当はすべて JSON ファイルに記述できるようにした。さらに。各照明機器に対してどのように振る舞うか/マイコンとどのように通信するか はフロントエンドとは独立した Python コードにし、実行時に動的 import できるようにすることで、機器構成の変更に対応しやすくした。実際に、任期後半に機器を 2 つ追加したが、Python コードを編集することなく対応することができました。
  3. 依存関係を自動的に解決するバッチファイルを作成した。そもそも Python プログラムはビルド不要で動作するわけだが、「Python のインストール・バージョンの確認」「必要なパッケージの自動インストール (pip)」「マイコンとの通信の自動設定」等を行う単一のバッチファイルを作成し、本アプリケーションを動作させるコンピュータが変更しても指示に従うだけで運用可能にできるようにしました。

保守性の向上(ハードウェア)

ソフトウェアだけでなくハードウェアでも、自分が居なくなった後何年も使用できるように工夫しました。

最初に作った制御装置部分は、Arduino Uno R3 と Arduino DMX Shieldを使用しました。Arduino Uno R3 は今後の入手性や初期開発の利便 (もともと Arduino には親しんでおり勝手知ったる)を考えての選択でしたが、これらには以下の問題がありました:

  1. Arduino Uno R3 は登場から 10年以上経ち、性能が陳腐化しているなか価格は高い。実際に今作では MCU の機能が不足しており、低速で安定性の低い SoftwareSerial を使用せざるをえなかった。
  2. 開発の約 1 年後、Arduino は Arduino Uno R4 をリリースした。使用しているプログラムが R3 に搭載の MCU に依存しており、R4 で動作するかが分からない。公式 R3 は今後入手性が低くなることが予想されるし、互換ボードは正しい選択が難しい。
  3. Arduino DMX Shield は 入手性が不安定 (日本代理店では頻繁に品切れ状態になる)。さらに、Arduino Uno 以外のフォームファクタには使いにくい形状をしている。

上記の問題を 対処/影響を軽減 するために、それぞれ以下のことを気をつけながら、回路やプログラムを再設計しました:

  1. Arduino の採用をやめ、Raspberry Pi Pico を採用した。Pico にはハードウェア UART が 2 系統あるし、そもそも PC との通信はネイティブ USB 1.1 を使用でき安定性が非常に高い。さらに、30倍以上の性能を持つが価格は 1/4 程度である。
  2. 2024年、Raspberry Pi Pico 2 が発表されたが、直接メモリマップにアクセスせずに C++ でプログラムを書く限り Pico (1) と高い互換性がある。さらに、Pico (1) も未だ公式品の入手が容易である。
  3. DMX 信号生成部分も、既製品を使うのではなく、LTC485 (RS485 トランシーバ IC) を使用した回路を新たに起こして実装した。RS485 トランシーバ IC は今後しばらくは入手可能なことが予想される。

マイコンを使用した PC レス サウンドアート・グラフィックアート装置

使用技術

  • C++ (Raspberry Pi Pico SDK、Arduino-Pico)
  • MicroPython (Raspberry Pi Pico)

製作期間

2025年8月～現在

概要

2025年9月26日、私はオーディオビジュアルイベント 「Flow vol.8」に、グループSUBMITのVJ として出演しました。出演にあたり、音を生成/音楽を再生する装置と、映像を生成する装置を Raspberry Pi Pico を用いて作成し、それを用いてパフォーマンスを行いました。

技術構成図

頑張ったこと

音の生成・映像の生成からプログラムで自作

本作では、「音楽をソフトウェアミキサに流して再生」「映像を Dx などの API に流す」だけでなく、「受け取った PCM データをどのようにアナログ音声として再生するか」「受け取った映像をどのように信号出力にするか」に関してもプログラムを記述しました。

  • 音では、ハードウェアへの出力には Arduino-Pico ボードに同梱されている PWMAudio ライブラリを使用し、ソフトウェアミキサ・音源の部分は C++ のコードを書いていちから実装しました。
    また、音源で再生する音楽データは MIDI ファイルに格納しましたが、SD カードから MIDI ファイルを読み込み、再生する部分のプログラムも作成しました。MIDI ファイルのパーサーも自作しました。
  • 映像では、ハードウェアへの出力はライブラリを使用し、Pico の PWM 出力を用いて NTSC 映像信号を直接生成しました。マイコン上でどのような映像を表示するかは、映像表現あたり一個ずつ C++ のプログラムを記述しました。
    また、映像生成ライブラリを改造して、「垂直同期信号を意図的に出力せず、画面を縦方向にバグらせる」「カラーバーストの周波数をわずかに変え、映像への着色をバグらせる」などのソフトウェア映像効果も実装し、パフォーマンスで使用しました。

概要

このWebページです。HTML と CSS を用いた静的 Web ページです。

頑張ったこと

HTML LS、CSS4 への準拠

このページを作成した主たる動機は、就活に際して自分の製作物をまとめておく必要があると感じたからです。しかし、私にとっては、「最新の HTML 規格と CSS 規格を再勉強し、よりスマートな静的Webサイトを作成する」というもう一つの目的もあります。

これまで私は、「ブラウザゲーがつくりたい！」「最新のフレームワークを勉強してみよう！」といったような動機からWebフロントエンド技術に触れ、その結果としてHTMLやCSSの知識は場当たり的に得てきました。ところが、最新の標準規格である「HTML LS」「CSS4」と自分の知識に乖離があり、きちんとした / そうあるべき Webページが書けていないことに気付きました。
アルバイトでもHTMLを書く機会があるため、このままでは良くないと、再度 HTML LS について勉強しようと思い、ひとつずつ MDN を参照しながら、どのように書くべきかを再学習していきました。

本ページは、w3c の HTML validation テストを通過しています。
